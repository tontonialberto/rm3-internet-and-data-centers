    redistribute connected
    network <network address>/<mask> // invia dati multicast RIP su tutte le interfacce che matchano questo prefisso

BGP

debug bgp keepalives
debug bgp updates in
debug bgp updates out

### IMPORTANTISSIMO!!!! ###
    router bgp <my as number>

Peering setup:
    neighbor <ip> remote-as <asn>
    neighbor <ip> description <text>

Per verificare che funziona:
- tail -f /var/log/frr/frr.log e cerca i messaggi BGP
- show ip bgp summary (vedi messaggi ricevuti e inviati)
- show ip bgp neighbors (fornisce più dettagli)
- show ip bgp (dovrebbero esserci 0 prefissi)

show ip bgp <network address> (mostra gli annunci di quella network e la best route selezionata)

Annunci:
    network <network address>/<netmask>
Nota:
    posso annunciare solo rotte che sono nella mia kernel routing table.
    Altrimenti posso usare (!!CONTESTUALI BGP!!): 
        no bgp network import-check
    
    Senza filtri "in", non accetto niente in ingresso.
    Senza filtri "out", non annuncio niente.
    Se non ho filtri in/out posso usare (contestuale bgp):
        no bgp ebgp-requires-policy

Filtraggio per prefisso:
    neighbor <ip> prefix-list <pl-name> in
    ip prefix-list <pl-name> [seq <N>] permit <network addr>/<netmask> [deny any]
Nota: metti prima le più specifiche e poi le più generali (any).
    Per ogni rotta, viene applicata la prima regola applicabile.

Filtraggio per as-path:
    neighbor <ip> filter-list myACL in
    ip as-path access-list myACL permit ^200_300

Relazioni customer-provider:
    - Stub: customer ha 1 link verso un solo provider
    - Multi homed stub: customer ha 2 link verso lo stesso provider
    - Multi homed: customer ha 2 link verso 2 provider

Ho 2 link l1 e l2. Per usare come backup l2 e in tutti gli altri casi l2 (lato customer):
    - Annuncio out (customer -> provider) aumento la metric su l2
    - Annuncio in (provider -> customer) diminuisco la local pref su l2
Default metric: 0
Default local pref: 100

route maps:
    neighbor <ip> route-map <name> <in|out>
    route-map <name> permit <seq-number>
        match ip address <acl-name|prefix-list <prefix-list-name>>
        set <metric|local-preference> <number>
    access-list <name> permit|deny <network/mask>

DNS: 

IN TUTTI I NS:
systemctl start named

nei pc: /etc/resolv.conf
    nameserver <default ipv4 of name server (use local ns if possible)>
    search <sottodominio>

negli authoritative name server: 
    /etc/bind/named.conf.options:
        options {
            directory "/var/cache/bind";
        };
    (PER I LOCAL NAME SERVER) /etc/bind/named.conf.options:
        options {
            directory "/var/cache/bind";
            allow-recursion { <ipv4/netmask della rete in cui cercare il NS primario>; };
            dnssec-validation no;
        };
    
    (ANCHE PER I LOCAL NS SERVE L'HINT A ROOT)
    /etc/bind/named.conf:
        include "/etc/bind/named.conf.options";
        
        zone "<sottodominio oppure .>" {
            type <hint|master>;
            file "/etc/bind/db.<sottodominio a rovescio oppure root>";
        };
    

    (PER TUTTI I NAME SERVER TRANNE ROOT) /etc/bind/db.root:
        .               IN      NS      ROOT-SERVER.
        ROOT-SERVER.    IN      A       <ipv4 del root assoluto>

    (per il dominio gestito da quel NS) /etc/bind/db.<sottodominio a rovescio>:
        $TTL 60000
        @   IN  SOA <FQDN del NS con punto finale> root.<FQDN del NS con punto finale> (
            2006031201 ; serial
            28 ; refresh
            14 ; retry
            3600000 ; expire
            0 ; negative cache ttl
        )

        @   IN  NS <FQDN del NS con punto finale>

        ; per ogni PC in quel dominio (compresi i local NS, e sè stesso)
        <nome pc>.<FQDN del NS con punto finale>    IN  A  <ipv4 del PC>

        ; per ogni NS per zone DIRETTAMENTE sottostanti a questo dominio
        <sottodominio con punto finale>     IN NS   <FQDN del NS primario, con punto finale>
        <FQDN del NS primario, con punto finale> IN A <il suo ipv4>
